Locked decisions

Auth
Stateless JWT
JWT carried in an HttpOnly cookie, no server session — easy to demo multiple roles at once
Tenancy
Multi-tenant
Client Orgs isolated from each other; TicketFlow1-side roles work across all
Priority
Display-only fixed set
LOW/MED/HIGH/CRITICAL — filterable, does not drive SLA
Status model
One status set + per-type maps
Transition legality lives in TicketTransitionService, not scattered
SLA
Computed at read time
No scheduled job; deadlines persisted, status derived on read
Attachments
Metadata-only (MVP)
Filename/type/size recorded; real bytes deferred
Out of scope
DISPUTE type
Type model designed to add it later without restructuring
Layering
controller → service → repo
DTOs at the boundary; entities never leak past the service

Data model

Organization

Client company. Client users & their tickets belong to one; TicketFlow1-side roles span all.

AppUser

Account + role. org required for CLIENT_*, null for TicketFlow1-side.

Ticket

Central record. type · status · priority · severity(defect) · responsibility · SLA fields.

ChangeProposal

Approve/reject gate. Change Requests only. Reused across resubmissions.

Comment

INTERNAL (TicketFlow1-only) or PUBLIC (client-visible).

Attachment

Metadata-only in MVP: filename, type, size, uploader.

AuditLog

Append-only. Every status/field change, comment, proposal decision.

StatusHistory

Append-only lifecycle timeline, separate from the audit feed.
